00218 /*RFC - TXT record Key=Value : The total size of a typical DNS-SD TXT record is intended to be small
00219    -- 200 bytes or less.
00220 
00221    6.6 Example TXT Record
00222    
00223       The TXT record below contains three syntactically valid key/value
00224       pairs. (The meaning of these key/value pairs, if any, would depend
00225       on the definitions pertaining to the service in question that is
00226       using them.)
00227    
00228            -------------------------------------------------------
00229            | 0x09 | key=value | 0x08 | paper=A4 | 0x07 | passreq |
00230            -------------------------------------------------------
00231 */
00232 /*
00233 Service Instance Name = <Instance> . <subtypename>._sub.<Service> . <Domain>
00234 */
00235 /*
00236 DNS
00237 1.      The maximum length of any dns label is 64 bytes (63 bytes of characters plus 1 byte for length field)
00238 2.      The maximum length of the fully qualified dns name shall be 255 bytes
00239 
00240 DNS-SD
00241 a.      Max length of instance part = 63 bytes.
00242 b.      Max len of sub type     = 69 bytes (63 bytes subtype + 1 byte subtype len + 4 bytes of _sub + 1 byte of _sub len)
00243  
00244 c.      Max len of  service type = 22 bytes (16 bytes of srvc type + 1 byte of servicetype len + 4 bytes of protocol + 1 byte of protocol len
00245 d.      Worst case num Bytes remained for domain name = (255 - (64+69+22)) = 155. We only support .local domain. So domain name len = 7 bytes.
00246 */
