PUBLIC GSN_STATUS GsnSsl_DataReceive ( VOID *  pSSLConn,
INT32  sockDes,
UINT8 **  pRxDataBuf,
UINT32 *  pRxDataLen,
INT32  timeOut 
)

SSL Receive API.

This function Receives the data on the given socket. Since SSL works on TCP, and SSL encryption/decryption is block by nature, the SSL decoing can only be called after recepit of a complete SSL packet. This API receives the complete SSL frame and decodes and gives the frame to user.

Parameters:
pSSLConn- IN Context of SSL module.
sockDes- IN Socket descriptor on which SSL frame will be received.
pRxDataBuf- OUT pointer to store the received data.
pRxDataLen- OUT Length of the received data.
timeOut- IN time in sec after which the receive process will be abandoned if it doesn't receive any packet on the given socket.
Return values:
GSN_SUCCESSif Operation successful, else GSN_FAILURE.
Note:
User has to free up the memory pointed by the pointer pRxDataBuf after using it.
PUBLIC INT32 GsnSsl_Decode ( VOID *  pSSLConn,

UINT8 **  pOutDataBuf,
UINT32 *  pOutDataLen,
INT32  sockDesc 
)

SSL Decode API.

This function decodes the data. User has to pass the input data pointer which needs to be decoded. This function does a memory allocation to store the decoded data and passes the pointer in pOutDataBuf. User has free to this memory once the decoded data is utilized.

Parameters:
pSSLConn- IN Context of SSL module.
pInDataBuf- IN Pointer to the input data which needs to be decoded.
inDataLen- IN Length of the input data.
pOutDataBuf- OUT pointer to store the decoded data buffer location.
pOutDataLen- OUT Length of the decoded data buffer.
sockDesc- IN The Socket Descriptor
Return values:
1if Operation successful, else -1. Operation may fail if the memory allocation fails, or the decoding fails.
Note:
User has to free up the memory pointed by the pointer pOutDataBuf after using it. User should call GsnSsl_Free to free up the memory, insted of gsn_free.
